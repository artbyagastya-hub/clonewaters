Artwork Delivery Guide

Checklist for preparing can label artwork

📐

File Format & Specs

  • Format: PNG (transparent) or SVG vector
  • Resolution: 300 DPI minimum
  • Color mode: CMYK for print, RGB for web
  • Max file size: 20MB per asset
  • Name: brand-release-asset (e.g. cw-teddyhop-label.png)
📏

Dimensions

  • Full wrap label: 330mm × 150mm (print)
  • Web label (flat): 800 × 1200px min
  • Can mockup (3D render): 1200 × 1800px
  • Thumbnail (product cards): 400 × 600px
  • Social share (OG image): 1200 × 630px
🎨

Color & Brand

  • Include hex codes for all primary colors
  • Brand logo: separate file, dark + light bg
  • Ensure text readable at can scale
  • Provide Pantone refs for print house
  • Background must work on dark (#08080c)
🐻

Character Assets

  • Bear character: isolated on transparent bg
  • Vector (SVG) preferred for scaling
  • Provide alternate poses if available
  • Sunglasses lens gradients as separate layer
  • Hops illustrations: isolated, transparent
✏️

Typography

  • Brand name "Clone Waters" as vector path
  • Beer name as vector path (e.g. "Teddy Hopper")
  • Provide .OTF/.TTF font files if custom
  • Spec sheet: font name, weight, size, color
  • ABV, style, volume text as separate layer
📋

Content Checklist

  • Beer name (English + Vietnamese)
  • Beer style (e.g. DIPA, Sour, Stout)
  • ABV percentage
  • Volume (330ml or 355ml)
  • Ingredients list (optional but recommended)
  • Hops varietals listed
  • IBU if applicable
  • "Keep Cold / Drink Fresh" tagline
  • Legal: "Contains alcohol 18+"
🖼️

Layered Source Files

  • Deliver layered PSD or AI file
  • Separate layers: bg, character, text, hops, effects
  • Smart objects for easy can mockup swap
  • Include clipping mask for can shape
  • Label wrap guide overlay (cylinder template)
✅

Pre-Delivery Check

  • Spell check all text (EN + VN)
  • Preview on actual can mockup (3D render)
  • Test at thumbnail size — still readable?
  • Check color contrast on dark background
  • Get sign-off from brand owner
  • Archive final files with version number
